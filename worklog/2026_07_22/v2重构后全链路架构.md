# PhysGenLoop V2

作者：lsh

PhysGenLoop V2 是一条面向视频生成物理一致性的闭环流水线：

```text
VideoPhy2 manifest / prompt
  -> Wan2.2 candidate generation
  -> SAM2 + Qwen3-VL critic
  -> CriticReport V2 codec
  -> mask_manifest.json / critic_report.json
  -> Acceptance Gate + PolicyGuard
  -> Repair Policy decision
  -> Executor: prompt_repair / global_regeneration / local_editing / reject
  -> Re-Critic / Re-Gate
  -> loop_result / repair_trace / trials / summary
```

本仓库已收敛为 V2 单链路。历史 `LoopController`、benchmark、数据生产、旧 Learning Repair 训练与 release 系统不再属于当前运行面。

## 入口

V2 主入口：

```bash
python agents/wanphysics/run_videophy2_loop_v2.py --dry-run
```

真实 GPU 链路入口：

```bash
python agents/wanphysics/run_videophy2_loop_v2.py \
  --enable \
  --allow-proxy-policy \
  --manifest evaluation/manifests/videophy2_smoke_dev10.json \
  --limit 1 \
  --max-rounds 2 \
  --output-root outputs
```

forced four-action trial 入口：

```bash
python agents/wanphysics/run_actual_trials_v2.py \
  --enable \
  --allow-proxy-policy \
  --manifest evaluation/manifests/videophy2_smoke20.json \
  --output-root outputs
```

## 当前边界

- V2 使用 Wan2.2 作为候选生成后端。
- Critic 路径为 SAM2 + Qwen3-VL，并通过 V2 codec 保留 violations、critical frames 和 evidence。
- Local Editing 只允许 strict `mask_manifest.json`，禁止全帧 fallback 和单 mask 复制。
- 当前 policy checkpoint 是 proxy research 模式，真实运行必须显式传 `--allow-proxy-policy`。
- HunyuanVideo 旧计划、Blender 数据生产和 benchmark 报告系统已从当前主链路移除。

## 工作方式

- 修改代码前先读当前 V2 架构文档和相关代码。
- 不触碰本地临时目录、远程同步脚本和备份目录。
- 删除、移动、批量改名属于高风险操作，执行前必须确认范围。
- Git 提交信息遵循 Conventional Commits，不写 AI/Claude/Co-Authored-By。
