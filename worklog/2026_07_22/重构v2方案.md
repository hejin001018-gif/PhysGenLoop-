# PhysGenLoop V2 单链路重构方案

日期：2026-07-22
作者：lsh
目标：以 `worklog/2026_07_21/全链路代码架构_v2.md` 为唯一目标架构，收敛本地代码到一条可运行、可审计、可测试的 WanPhysics V2 全链路。

## 0. 结论先行

当前仓库同时保留了多套历史闭环、benchmark、Learning Repair、数据生产、远程同步脚本和临时备份。V2 真正需要的主链路已经明确：

```text
VideoPhy2 manifest / prompt
  -> Wan2.2 candidate generation
  -> SAM2 + Qwen3-VL critic
  -> CriticReport V2 codec
  -> mask_manifest.json / critic_report.json
  -> Acceptance Gate + PolicyGuard
  -> Repair Policy decision
  -> Executor: prompt_repair / global_regeneration / local_editing / reject
  -> Re-Critic / Re-Gate
  -> loop_result / repair_trace / trials / summary
```

重构原则：

- 只保留一条 V2 运行入口：`agents/wanphysics/run_videophy2_loop_v2.py`。
- 只保留一个真实 Trial 采集入口：`agents/wanphysics/run_actual_trials_v2.py`。
- V2 sidecar 作为核心实现：`generators/wanphysics/v2/`。
- 保留 V2 依赖的最小契约层，不把所有历史 `physgenloop` / `learning_repair` 功能直接删光。
- 临时文件、备份目录、远程辅助脚本不参与重构、不提交、不移动、不删除。
- 删除、移动、批量改名属于危险操作，执行前必须单独确认。

## 2. V2 必须保留的核心代码

必须保留：

