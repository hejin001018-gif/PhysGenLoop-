# Critic 基准测试迭代实施计划

**目标：** 生成一份可审计、留出的 VideoPhy-2 结果，其中基于 SAM2 的 PAVG 评判器在性能上显著优于匹配的直接 VLM 基线，同时记录失败的迭代而非隐藏它们。

**架构：** 将 SAM2.1 Hiera B+ 作为强制性密集观测前端。分离观测质量、确定性规则适用性、可选关键帧 VLM 验证以及模型主干效应，以便将改进归因于特定组件。在冻结的诊断分割上调整架构；仅对每个冻结的候选者在留出分割上评估一次。

**技术栈：** Python 3.12、PyTorch 2.13 CUDA 13.2、官方 Meta SAM2.1 Hiera B+、OpenCV、pytest、OpenAI 兼容聊天 API、VideoPhy-2、可选通过 SSH 远程 NVIDIA GPU。

---

## 实验契约

- 主要比较是匹配主干的：对于 D0/D1 以及每个调用 VLM 的 PAVG 组件，使用相同的模型家族。
- SAM2 对于 B1–M5 是强制性的。仅当作为明确命名的前端消融实验时，才允许稀疏 VLM 观测。
- 20 个视频的烟雾集仍作为管道/诊断结果，而非基准声明。
- 烟雾集确定性地分为 `dev10` 和 `eval10`，尽可能按物理标签和生成器分层。架构和阈值可以检查 `dev10`；`eval10` 在架构冻结后仅运行一次。
- 对于此烟雾门控，只有当留出 PAVG 宏 F1 至少达到 `D0 + 0.10`，平衡准确率至少达到 `D0 + 0.10`，失败率不更高，且两个类别召回率均非零时，结果才被视为“显著更好”。
- 观测到预测后不得移除任何样本。未知和失败的预测保留在分母中。
- 最多两个架构修订可以使用 `dev10`。模型升级在架构冻结后评估；它不得更改样本成员资格或阈值。
- 如果未满足这些门控，请如实报告负面结果，并将下一个决策移至更大的预注册服务器运行。在出现正数之前，不得在 `eval10` 上调参。
