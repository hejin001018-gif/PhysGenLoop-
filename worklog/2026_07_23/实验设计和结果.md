# PhysGenLoop V2 Strict Enforce 实验设计与结果

用途：PPT 文案稿。本文只总结 2026-07-22 最新 V2 主链路与 `v2_pilot300_real_limit50_strict` 的 50 条严格实验结果。

作者：lsh

## 1. 实验目的

验证 PhysGenLoop V2 是否能以一条干净、可审计的闭环完成真实视频生成后的物理评价与定向修复。

本次实验不追求放宽标准下的表面通过率，而是验证 Strict Enforce Gate 在真实链路中的 fail-closed 行为：只有物理、语义、质量、置信度、覆盖率等条件同时满足，样本才记为 `ACCEPTED`；否则进入修复、拒绝或评价不可用收口。

## 2. 实验对象与运行设置

- 数据来源：`evaluation/manifests/videophy2_pilot300.json`
- 实验规模：50 条样本
- 运行目录：`outputs/v2_pilot300_real_limit50_strict`
- 代码版本：`569ffb7e8ec61e6af1355737e6a1acc346101830`
- 最大轮数：4 轮修复闭环
- 接受模式：`acceptance_mode=enforce`
- Critic 配置：`sam2_seeded_rules`
- Policy 动作空间：`prompt_repair`、`local_editing`、`reject`
- 决策来源：`three_action_policy`

## 3. 链路设计

V2 主链路按统一标准执行：

1. Wan 生成初始候选视频。
2. Physics Critic 对视频构建物理证据，输出物理分数、置信度、覆盖率、违规信息等。
3. Strict Enforce Gate 做三态判定：`ACCEPTED`、`REJECTED`、`UNAVAILABLE`。
4. 只有 `REJECTED` 样本进入三动作 Policy。
5. Policy 选择 `prompt_repair`、`local_editing` 或 `reject`。
6. Executor 执行动作；其中 `local_editing` 通过 ProPainter 严格局部编辑实现。
7. 修复产物重新进入同一 Critic/Gate 标准，只有 Re-Gate `ACCEPTED` 才算严格成功。
8. 每个样本输出 `sample_status`、`loop_result`、`repair_trace`、`trials` 等审计文件。

关键约束：Executor 产物不等于成功；`reject` 是审计式终止，不是程序失败；`UNAVAILABLE` 是评价不可用收口，不混同为视频不合格。

## 4. 评价标准

Strict Gate 不是单一物理分数阈值，而是多条件同时约束：

- physics：物理合理性分数
- confidence：Critic 对判断的置信度
- coverage：物理证据覆盖率
- semantic：候选视频与当前 prompt 的语义一致性
- original_prompt_semantic：候选视频与原始任务语义一致性
- quality：视频质量分数
- blocking violations：是否存在阻断型物理违规

因此，即使某个候选的 `physics_score=1.0`，只要 confidence 或 coverage 不足，仍会被严格拒绝。

| policy 选择 `reject` | 24 |
| 实际执行 `local_editing` | 25 |
| 实际执行 `reject` | 43 |
| Guard allowed | 49 |
| Guard blocked | 19 |

Guard 拦截原因：

| blocked reason | 数量 |
| --- | ---: |
| `local_editing_requires_valid_mask_manifest` | 17 |

ProPainter 局部编辑进入了真实链路，但被严格约束在合法 mask manifest 和 local scope 内。Policy 想局部编辑时，如果 mask 不合法或修复范围过宽，Guard 会 fail-closed，直接转为审计式 reject。

结论：当前 ProPainter 分支工程上可执行，但有效修复比例仍低。多数局部编辑没有改变 Critic 物理判断，少数样本物理分明显提升，但仍被 coverage 或 confidence 拦下。

## 8. 典型案例

### 案例 A：物理分高但严格拒绝

- sample：`videophy2-00111bf7402bca39b0c3`
- prompt：A person carries a heavy bucket while wading through chest-deep water.
- best physics：1.0
- final state：`REJECTED`
- Gate reasons：`confidence_below_threshold`、`coverage_below_threshold`

说明：Strict Gate 不接受单一高物理分。证据覆盖率和置信度不足时，系统保持拒绝。

### 案例 B：Policy 想修，但 Guard 拦截

- sample：`videophy2-033609ae4d630a7095bc`
- prompt：Time-lapse of a pencil being sharpened with sandpaper; the pencil point becomes increasingly sharp.
- policy action：`local_editing`
- executed action：`reject`
- terminal reason：`local_editing_requires_valid_mask_manifest`

说明：局部编辑必须依赖合法 mask manifest。mask 不合法时不做不受控编辑，直接 fail-closed。

### 案例 C：多轮局部编辑仍未通过

- sample：`videophy2-072eb5a79d008def9fcd`
- prompt：A large pumpkin is placed on a small, flimsy stool; the stool breaks, and the pumpkin rolls onto the ground.
- final state：`MAX_ROUNDS`
- rounds：4
- 执行动作：多轮 `local_editing`
- best physics：0.177

说明：链路支持多轮 ProPainter 局部编辑与 Re-Critic/Re-Gate，但当前修复对物理证据没有形成有效改善。

### 案例 D：局部编辑提升物理分但仍未严格通过

- sample：`videophy2-1f2f0e39f797fa1aa9f4`
- prompt：Someone removes a large tarp, revealing a gleaming new car underneath.
- physics：0.177 -> 1.0
- semantic：0.90 -> 0.95
- after gate：`REJECTED`
- remaining reasons：`confidence_below_threshold`、`coverage_below_threshold`

说明：局部编辑可以改善部分物理指标，但 Strict Gate 仍要求评价证据足够完整。

## 9. 主要结论

1. V2 全链路工程上已跑通：生成、Critic、Gate、Policy、Guard、ProPainter Local Editing、Re-Critic、审计记录均有真实产物。
2. Strict Gate 行为符合设计：没有把评价证据不足的样本误记为成功，`accepted=0` 是严格门禁下的真实结果。
3. 当前最大瓶颈不是执行失败，而是评价与修复有效性：coverage 固定偏低、多门槛同时约束、mask manifest 合法性不足、局部编辑改善有限。
4. `reject` 应解释为系统可审计地拒绝样本，不是程序异常；`evaluation_failed` 是评价不可用收口，也不等同于视频物理失败。
5. 这次实验给出了下一阶段优化方向：先提升可评价覆盖率和局部编辑可执行条件，再看 accepted 是否增长。

## 10. 下一步实验计划

- 提升 Critic coverage：补齐事件声明、物体轨迹、关键帧证据，使 coverage 不再固定卡在 0.43。
- 改进 mask manifest 生成与校验：提高 `local_editing` 进入 ProPainter 的合法比例。
- 区分局部可修与全局不可修：减少 scope 为 broad 的样本误入局部编辑。
- 对 physics delta 为正的样本做案例复盘：分析为什么物理分提升后仍卡在 confidence/coverage。
- 在覆盖率修复后复跑 pilot50，再扩展到 pilot300，报告 accepted、max_rounds、evaluation_failed 和局部编辑成功率变化。
