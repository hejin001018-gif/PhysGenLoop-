### 2.2 主流水线（`pipeline.py` · 427 行）

固定执行顺序：

```
CriticRequest
  └─▶ PhysicsPlanResolver.resolve()           # Planner：Prompt → PhysicsPlan
      └─▶ replace(request, physics_plan=…)     # 后续所有模块看到同一份 plan
  └─▶ QuestionGraphGenerator.generate()       # 模板图 / Hybrid(模板+PQSG)
  └─▶ _observe_video()                        # OpenCV 解码 → detector → tracker
  └─▶ TrajectoryExtractor.extract()           # 位置/速度/加速度 + floor_y 平滑
  └─▶ EventDetector.detect()                  # 接触/反弹/消失/穿透/瞬移
  └─▶ MechanicsEvaluator.evaluate()           # 力学一致性（可选）
  └─▶ PhysicsRuleEngine.evaluate()            # 确定性规则候选
  └─▶ TemporalLocalizer.localize()            # 每个候选定位 start/peak/end
  └─▶ VisualEvidenceExtractor.extract()       # 外部 CV 插件（可选）
  └─▶ VideoScienceChecklistEvaluator.evaluate # 五维证据检查表（可选）
  └─▶ KeyframeSelector.select()               # 候选 × 关键帧
  └─▶ QuestionGraphExecutor.execute()         # DAG 路由到观察/事件/规则
  └─▶ VLMVerifier.verify_many() / verify()    # 关键帧 VLM 复核（可选）
  └─▶ ResultFusion.fuse()                     # 单候选综合分
  └─▶ QuestionGraphScorer.enrich_report()     # 图节点分数并入 report
  └─▶ CoverageAwareEvidenceFusion.enrich()    # 五族权重覆盖感知融合
  └─▶ CriticArtifacts / CriticReport
```

## 三、physgenloop 闭环编排包

### 3.1 公开 API（`src/physgenloop/__init__.py`）

```python
from physgenloop import (
    LoopController, LoopConfig, LoopResult, LoopRound,
    CandidateEvaluation, GeneratedCandidate,
    DeterministicFakeGenerator,    # 当前占位生成器（不写 MP4）
    PhysicsCriticAdapter,           # 桥接 pavg_critic.PhysicsCritic
    InstructionPromptRepairer,      # 追加 repair_instruction 到 prompt
    EvidenceAwareSelector,          # decision > physics_score > confidence
)
```

### 3.2 冻结契约（`contracts.py` · 71 行）

- `LoopConfig(max_rounds=3, candidates_per_round=2, acceptance_score=0.8, base_seed=42)`
- `GeneratedCandidate(candidate_id, video_path, prompt, seed, metadata)` — 视频引用而非视频本体
- `CandidateEvaluation(candidate, report)` — 候选 + Critic 报告
- `LoopRound(round_index, prompt, evaluations, selected_candidate_id)` — 单轮审计记录
- `LoopResult(best, history, stop_reason, resolved_plan)` — 最终结果 + 完整历史

### 3.3 Protocol（`interfaces.py` · 40 行）

- `VideoGenerator.generate(prompt, physics_plan, seed) → GeneratedCandidate`
- `PlanResolver.resolve(request) → PhysicsPlanResolution`
- `CandidateCritic.evaluate(candidate, prompt, physics_plan) → CriticReport`
- `PromptRepairer.repair(prompt, report) → str`
- `CandidateSelector.select(evaluations) → CandidateEvaluation`

任何后端只要实现对应 Protocol 即可插入 LoopController，主链路不依赖具体实现。

### 3.4 LoopController（`controller.py` · 108 行）

单次调用 `run(prompt, physics_plan=None)`：

```
1. PlanResolver 把 prompt+plan 解析为 resolved_plan
2. for round in range(max_rounds):
     for k in range(candidates_per_round):
         seed = base_seed + round*K + k
         cand = generator.generate(prompt, plan, seed)
         report = critic.evaluate(cand, prompt, plan)
         evaluations.append((cand, report))
     selected = selector.select(evaluations)
     if selected.decision == "physical" and score >= acceptance:
         return LoopResult(best=selected, stop_reason="accepted")
     prompt = repairer.repair(prompt, selected.report)
3. return LoopResult(best=selector.select(all_winners), stop_reason="max_rounds")
```

### 3.5 当前实现

- `generator.py` · **DeterministicFakeGenerator**：不写 MP4，返回 `fake://{sha}.mp4` 占位。真实 HunyuanVideo 后端待接入。
- `critic_adapter.py` · **PhysicsCriticAdapter**：把 `GeneratedCandidate` 包装为 `CriticRequest` 转发给 `pavg_critic.PhysicsCritic`。
- `repairer.py` · **InstructionPromptRepairer**：去重后 `prompt + "\nPhysics correction: " + " ".join(instructions)`。
- `selector.py` · **EvidenceAwareSelector**：三级排序 `(decision_rank, physics_score, confidence)`，`decision_rank` 优先 `physical > unknown > violation`。

## 四、agents 探索层（独立于 physgenloop）

`agents/` 保留为 LLM 改写与多后端探针，不进入 `LoopController` 主链路。

| 文件 | 行数 | 职责 |
|------|------|------|
| `prompt_rewriter.py` | 141 | Claude / OpenAI / Stub 三 backend 的 prompt 改写层 |
| `repairer.py` | 146 | Repair 决策器（L1 `prompt_only` / L2 `local_inpaint`(disabled) / L3 `full_regen` / `stop`）+ generator_request 构建器 |
| `video_backend.py` | 152 | `hunyuan-local` / `replicate` / `fal` / `stub` 抽象 |

Repair 决策规则：

| physics_score | 决策 |
|---------------|------|
| `> 0.6` | `prompt_only`（改 prompt，沿用旧 seed） |
| `(0.4, 0.6]` | `local_inpaint`（首期关闭，等 HunyuanVideo mask 支持） |
| `≤ 0.4` | `full_regen`（换 seed + 强化 prompt） |
| `is_physical=True` / 达轮数上限 / 连续 2 轮无提升 | `stop` |

与 `physgenloop.InstructionPromptRepairer` 的差异：`agents.repairer` 输出完整 `generator_request` dict，`physgenloop` 版只做 prompt 拼接。**当前两条修复路径共存，未合并**。

### 5.1 JSON Schema

- `schemas/sample.schema.json` — 单样本标注（v1.0）
- `schemas/critic_output.schema.json` — Critic 输出（**v2.0**，三态 `decision` + `coverage` + `score_breakdown` + `node_results` + `evidence_bundles`）

### 5.2 配置

`configs/default.yaml` 覆盖：

- `detector`：HSV 红球阈值 + `floor_y_ratio`
- `tracker`：`max_match_distance_px` / `max_missed_frames`
- `trajectory`：`smoothing_window`
- `events`：速度阈值、接触/穿透容差、消失/上升最小帧数、瞬移速度
- `rules`：`enabled = [premature_rebound, surface_penetration, object_disappearance, reverse_gravity, teleportation]`
- `temporal`：pre/post context 帧数
- `question_graph` / `checklist` / `mechanics`：开关 + 置信度阈值
- `fusion`：五族权重 `rule=0.35 / pqsg=0.2 / checklist=0.2 / mechanics=0.2 / vlm=0.05`，`minimum_coverage=0.35`
