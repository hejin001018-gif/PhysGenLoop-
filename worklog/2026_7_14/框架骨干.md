## 一、判词

**外域十脉可 100% 替代 PDF 原设「Blender-only 自造数据 + 自研 Critic 基线」路径。**
新框架应以：

- **Kubric** 替 Blender MCP 自动生成环节；
- **Physion + IntPhys 2** 替首期异常样本自造；
- **VideoPhy2 + PhyGenBench + Physics-IQ** 三合一替单一评测；
- **VideoRepair 思想** 注入 Repair Agent。

原三阶段架构骨架不变，但**数据源、评测矩阵、Repair 策略**三处必须升级——否则重复造轮子。

## 四、L1–L2 数据层：三源合流

### 数据源矩阵

| 数据源 | 定位 | 覆盖异常 | 真值粒度 | 用途 |
|--------|------|----------|----------|------|
| **Kubric**（Google + Bullet + Blender 渲染） | 主力自造管道，替代 PDF 原设 Blender MCP | 全部可控注入 | trajectory / contacts / bbox / mask / depth / flow / 3D 姿态 | 训练主集、参考仿真回滚 |
| **Physion v1/v2**（ThreeDWorld） | 8 类场景现成集：碰撞、下落、滚动、包容、多米诺、悬挂、弹拽、支撑 | 主要为正常物理 + OCP 对照 | 3D 状态真值 | 训练增广、跨引擎泛化测试 |
| **IntPhys 1 & 2** | 异常配对基准（正常/违反成对） | 精准对齐 PDF 首期 4 类：重力/碰撞/穿透/恒存 | 二元配对 + 帧区间 | Critic 二分类基线、异常检测评测 |
| **HunyuanVideo 错误采集** | 生成模型自身错误分布 | 长尾真实错误 | 需 Critic + 人工复核 | 域适配、闭环回归集 |
| **少量真实视频** | 泛化护栏 | 未知分布 | 人工标注 | Zero-shot 测试 |

### 关键决策

- **Kubric 成为一等公民数据管道**。README 提到的 "Blender + Blender Python API" 保留为备选（视觉更真实但工程化差）。
- **Schema 对齐**：Kubric 输出格式与 README `trajectory.json / contacts.json / annotation.json` 语义无缝对齐，只需一层适配器。
- **切分策略**：训练/验证/测试按 **场景模板 + 对象组合 + 随机种子** 分组切分，避免相似场景泄漏（README § 数据来源 已规约）。

## 五、L3 Physics Critic 层：混合式增强

保留 README 原设主链路：

```
Det/Seg → Track → Trajectory → Event → Rule ⊕ VLM
```

### 新增两条增强

**增强 1：参考仿真回滚（Reference Simulation Rollback）**

- 从视频 T=0 反解初始状态（对象位置、速度、质量估计）。
- 投喂 Kubric 复现「应有物理」轨迹。
- 实际轨迹 vs 参考轨迹 → 逐帧偏差分数。
- README 中 `reference_simulation` 字段（当前 `null`）由此填充。

**增强 2：双路 VLM Critic**

- **同源 VLM**（与生成模型同族）：捕捉生成模型自身偏差。
- **异源 VLM**（Qwen2.5-VL / InternVL）：避免「看不见自己错误」陷阱。
- 双路分歧样本 → 强制人工复核（README § 质量控制 已规约）。

### 评分融合

```
physics_score = w1·rule + w2·refsim + w3·vlm_same + w4·vlm_diff
```

权重由 **IntPhys 2 校准集** 学出。

## 六、L4 Agentic 闭环层：VideoRepair 思想注入

原 README 闭环：

```
Planner → Generator → Critic → Repair → Regeneration → Selector
```

### 升级点：Repair Agent 四级修复策略

| 层级 | 触发条件 | 手段 |
|------|----------|------|
| L1 Prompt 增强 | 轻度违规 (score > 0.6) | 追加 `physics_hint` 字段（如 "ball must contact floor before rebound"） |
| L2 局部区域重生成 | 单对象违规、时序局部 | 取 `critical_frames` 邻域帧 + mask，做局部 video inpainting |
| L3 全片重生 | 严重违规、多对象耦合 | 换 seed + 强化 Planner 约束，Best-of-K 兜底 |
| L4 提前终止 | K 轮无改善 | Selector 从历史候选取最优，落地 `stop_reason` |

### Selector 分数

整合 **VideoPhy2 + PhyGenBench + Physics-IQ** 三重视角 + 语义保持（CLIP-Score）+ 画质（VBench 子项）加权。

## 七、L5 评测矩阵层：五合一测评

| 基准 | 测评侧重 | 用于 |
|------|----------|------|
| Physion Eval | 8 类场景物理理解 | Critic 分类准确率 |
| IntPhys 2 | 异常配对 AUC / F1 | Critic 二分类 & 定位 |
| VideoPhy2 | 视频是否符合物理定律 | 端到端生成质量 |
| PhyGenBench | 物理生成综合基准 | 端到端生成质量 |
| Physics-IQ | 物理智商 | 端到端生成质量 |
| VideoScience-Bench | 力学评估框架 | Selector 权重校准 |

### 报告主指标

- **Critic 侧**：Physion + IntPhys 2 (AUC / F1)。
- **生成侧**：VideoPhy2 + PhyGenBench + Physics-IQ 三均值。
- **禁止**：单一均值掩盖冲突——分开报告。

## 八、验尸：三关校验

| 关卡 | 判定标准 |
|------|----------|
| **数据关** | Kubric 输出的 `trajectory.json` 能直接喂给 Critic Rule Engine，无需二次转换 → schema 打通 |
| **Critic 关** | 在 IntPhys 2 dev set 上 **AUC ≥ 0.80**，即可宣称能识别 4 类核心异常 |
| **闭环关** | 在 100 条 HunyuanVideo 生成的物理违规样本上，多轮反馈相对单次生成 **VideoPhy2 分数提升 ≥ 15%** → 回答 PDF 核心研究问题 3、4 |

## 九、余劫：五处道基裂痕

| # | 风险 | 位置 | 兜底 |
|---|------|------|------|
| 1 | Kubric ↔ HunyuanVideo 域差异过大 | L1-L2 | 混入 Physion + 生成模型输出双向域适配 |
| 2 | 参考仿真反解初始状态失败（单目 3D 歧义） | L3 增强路径 | 仅深度可靠场景启用，否则退化为规则+VLM |
| 3 | VideoPhy2/PhyGenBench 与 IntPhys 2 结论冲突 | L5 | 分开报告，不做单一均值 |
| 4 | Repair 局部重生导致语义漂移 | L4 | Selector 硬约束语义保持 ≥ 阈值，否则回退上一轮最优 |
| 5 | HunyuanVideo-1.5 显存/推理时长瓶颈 | L1 引擎 | 首期 480p + K=2 + 最多 2 轮反馈 |
