## 三、`pavg_critic` 主要提升对比

### 3.1 Qwen3 / vLLM 结构化输出兼容

涉及：

- `src/pavg_critic/api_models.py`

提升：

- `OpenAIChatModel` 新增可选 `strict_json_schema`；
- 默认继续使用兼容性更广的 `json_object`；
- 对支持严格 schema 的 Qwen3/vLLM 服务，可发送 `response_format.type=json_schema`；
- 纯文本与多模态调用使用同一 response-format 构造逻辑；
- 不改变默认调用行为，避免破坏旧 OpenAI-compatible provider。

价值：减少 Qwen3 返回字段缺失、类型漂移或附加无关字段导致的解析失败，同时保留旧服务兼容性。

### 3.2 VideoPhy 数据清洗与 pilot 抽样

涉及：

- `src/pavg_critic/benchmarking/datasets.py`

提升：

- 将空缺 `prompt_group_id` 统一映射为稳定占位组；
- 忽略物理规则列中的 `nan` / `[nan]`，避免将缺失值当作真实规则；
- 新增确定性的 action-covering pilot 抽样，在 action 覆盖基础上平衡 label、generator 和 label×generator；
- 下载 URL 在请求前对路径进行安全编码，修复视频 URL 含空格或特殊字符时的下载失败；
- 保持 source row、sample ID 和最终清单可复现。

价值：让 300-sample pilot 更接近冻结设计中的 action/generator/label 分层要求，并降低数据清洗造成的伪标签和下载失败。

注意：pilot 子命令和 CLI 参数位于 `benchmarks/prepare_videophy_manifest.py`。若后续只同步 `src/pavg_critic` 而不更新 CLI，新抽样函数会存在但不会由旧命令入口直接暴露。

### 3.3 SAM2 首帧种子健壮性

涉及：

- `src/pavg_critic/sam2_detector.py`

提升：

- 在 JSON Schema 中约束 `x_pct/y_pct` 为 `[0, 100]`；
- 对模型仍可能产生的越界有限值执行 clamp；
- 显式拒绝 `NaN`、`+Inf` 和 `-Inf`；
- 将 100% 坐标限制到实际像素上界 `width-1/height-1`，避免点提示落在图像边界之外。

价值：减少 Qwen3 目标种子轻微越界导致整段 SAM2 初始化失败，同时不接受不可审计的非有限坐标。

### 3.4 M4 VLM 复核加入 SAM2 轨迹证据

涉及：

- `src/pavg_critic/vlm_verifier.py`
- `src/pavg_critic/pipeline.py`
- `src/pavg_critic/schemas.py`
- `src/pavg_critic/fusion.py`
- `src/pavg_critic/benchmarking/pavg_methods.py`

提升：

- 为候选附加有界、按时间排序的 SAM2 track snapshot；
- 长轨迹最多均匀保留 24 个状态，控制 prompt 体积；
- M4 不再仅按 violation category 合并候选，而是按 `object + category + start_frame + end_frame` 分组；
- 避免不同对象或不同时间段的同类异常共享一条 VLM 结论；
- VLM 输出新增 `confirmed/rejected/uncertain` 三态 `claim_status`，旧 provider 缺省为 `uncertain`；
- prompt 明确要求只判断当前证据段、拒绝 tracking artifact，同时不得把生成 prompt 明确要求发生的事件误判为异常；
- `claim_status` 保留到最终 violation evidence，便于审计；
- M4 默认 detector/VLM 权重由 `0.4/0.6` 调整为 `0.7/0.3`，避免开发集上 VLM 将所有规则候选一律否决。

价值：M4 的判断对象从“抽象类别”收紧为“具体对象在具体时间段的候选”，并让 VLM 看到形成规则候选的轨迹证据，而不是只看少量关键帧。

### 3.5 B1 规则基线 payload 隔离

涉及：

- `src/pavg_critic/pipeline.py`

提升：

- 先统一完成候选时序定位；
- 仅在启用真实 VLM verifier 时附加 `sam2_track` evidence；
- `NoOpVLMVerifier` 对应的 B1/M1/M2/M3 不附加只供模型消费的扩展 payload。

价值：保持 B1 是纯规则 baseline，避免报告体积增加、哈希变化或 benchmark 因无关模型 payload 而失去可比性。

### 3.6 全量报告基础指标

涉及：

- `src/pavg_critic/benchmarking/metrics.py`

提升：

- 新增 `physical_recall`，与已有 `violation_recall` 对称；
- 新增无第三方依赖的线性 percentile；
- 输出 `p50_latency_sec` 和 `p95_latency_sec`；
- 保留原有 accuracy、balanced accuracy、Macro-F1、unknown/failure rate 和 Spearman 语义。

价值：满足 full VideoPhy 报告对两个类别召回率和尾部延迟的预注册要求，避免均值掩盖类别塌缩或长尾时延。
