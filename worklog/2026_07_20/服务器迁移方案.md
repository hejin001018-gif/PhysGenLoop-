当前状态：`main ...origin/main [ahead 134, behind 3]`，远程 `origin=https://github.com/hejin001018-gif/PhysGenLoop-.git`。**推之前必须消除分叉**，否则新机拉的历史不全。

```bash

git ls-files | grep -iE '\.(mp4|safetensors|pt|bin)$'   # 应为空
```

## 五、方式对比：分通道 vs 整机直传

本方案主推**分通道**（GitHub+HF+官方源），原因：代码有版本管理、官方模型省 49G 往返、资产可复用。

若两台服务器网络互通且想省事，可整机 `rsync` 直传（跳过 HF）：

```bash
# 在新机执行（示例，按需保留/排除）
rsync -avP --exclude='envs' --exclude='outputs/benchmarks' \
  -e "ssh -p 27323" root@px-cloud1.matpool.com:/root/PhysGenLoop-/ /root/PhysGenLoop-/
```

- 优点：一条命令原样复刻（含 models/data/checkpoints/outputs）。
- 缺点：需两机直连、49G+ 传输慢、代码不进 GitHub 版本库。
- `envs/` 仍建议排除后重建（跨机二进制不可移植）。

> 即便走 rsync，**代码仍建议照第二部分推一份到 GitHub**，保留团队版本线。
