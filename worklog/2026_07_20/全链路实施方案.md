# 全链路实施方案：videophy2 → Wan2.2 → 打分 → 全局/局部修复 → 达标输出

作者：lsh
> 日期：2026-07-20
> 目标：把已有模块串成完整闭环——给定 videophy2 prompt，Wan2.2 生成视频，打分模块判别物理合理性；全局错误用 repair agent 改 prompt 重生成，局部错误用 SAM 抽帧局部修复；循环校验直至达标输出。
> 前置：各模块已存在（Wan2.2 生成器、PAVG Critic、SAM2 检测器、ProPainter 局部编辑器、四动作 ExecutorRegistry、LoopController）。本方案只做「串链路」与「补关键缺口」。

---

## 一、目标全链路

```text
videophy2 CSV/manifest（caption 或 upsampled_caption）
        │
        ▼
  WanSubprocessGenerator（Wan2.2 生成视频）
        │
        ▼
  Sam2VlmSubprocessCritic（SAM2+VLM 打分 → CriticReport）
        │
        ▼
  达标？(decision==physical 且 physics_score>=acceptance_score)
   ├─ 是 → 输出合规视频，stop_reason="accepted"
   └─ 否 → 判定错误范围（帧占比启发式）
            ├─ 全局错误（critical_frames 占比高 / 无帧级证据）
            │     → repair agent 改 prompt → 全局重生成（GLOBAL_REGENERATION / PROMPT_REPAIR）
            └─ 局部错误（少数帧不符）
                  → SAM 抽异常帧 + mask → ProPainter 局部修复（LOCAL_EDITING）
        │
        ▼
  重新打分，循环（max_rounds 上限），直至达标或耗尽轮次
```

对应现有代码：`LoopController.run()` 已实现四动作分发骨架，本方案补齐「判定错误范围」和「SAM mask 落盘」两个关键缺口，并新增 videophy2 入口。

## 三、实施任务（按依赖顺序）

### 任务 1（G1）：打开 LOCAL_EDITING 能力位

**文件：** `generators/wanphysics/repairer.py`

- `ActionValueRepairer.__init__` 新增 `local_editor_available: bool = False` 参数
- `repair_with_decision()` 里 `RepairContext(local_editor_available=self._local_editor_available, ...)`
- `load_action_value_repairer()` 新增同名参数透传
- `controller.py` / `executor_factory.py`：注入了 executor_registry 时置 `local_editor_available=True`

**验证：** 构造带 local_editing 能力的 repairer，喂一个含 critical_frames 的 report，确认 DecisionPolicy 不再被 capability mask 屏蔽 LOCAL_EDITING。

**原则：** 这是四动作闭环真正生效的前提，不打开这个位，局部修复分支是死代码。

---

### 任务 2（G3）：SAM2 mask 落盘 + 写入 violation evidence

**文件：** `src/pavg_critic/sam2_detector.py`、`agents/wanphysics/eval_step.py`

- SAM2 检测已有每帧 mask（`out_mask_logits` → 二值 mask），当前只取 bbox。新增可选 `mask_output_dir`：为每个被判违规的对象/帧保存 mask PNG（`{object}_{frame:05d}.png`），返回 mask 目录路径
- `eval_step.py` 把 SAM2 输出的 mask 路径填入对应 `Violation.evidence["mask_uri"]`（单帧或帧序列的代表 mask）
- 保持向后兼容：不传 `mask_output_dir` 时行为不变（只出 bbox）

**验证：** 对一个已知局部违规视频跑 eval_step，确认 violation.evidence 里有 mask_uri 且文件存在、可被 cv2 读取。

**原则（KISS）：** mask 落盘只在需要局部修复时做，不预先为所有帧生成，控制开销。

---

### 任务 3（G2）：帧占比启发式判定（全局 vs 局部）

**文件：** 新增 `generators/wanphysics/error_scope.py`；接入 `controller.py`

- 实现 `classify_error_scope(report, total_frames, local_threshold=0.4) -> "global" | "local"`：
  - 聚合 report 所有 violation 的 critical_frames 去重
  - `ratio = len(critical_frames) / total_frames`
  - `ratio == 0`（无帧级证据）或 `ratio >= local_threshold` → `"global"`
  - `0 < ratio < local_threshold` → `"local"`
- `controller.py` 在拿到 DecisionPolicy 的 `decision` 后，用该启发式**校正/兜底**动作：
  - 判定 `"local"` 且 executor_registry 有 local_editing 且 report 有可用 mask/critical_frames → 强制走 LOCAL_EDITING
  - 判定 `"global"` → 走 GLOBAL_REGENERATION 或 PROMPT_REPAIR（保留 Policy 在两者间的选择）
  - `total_frames` 从 candidate.metadata 或 config.num_frames 取（loop.yaml=81）
- `local_threshold` 提到 `configs/loop.yaml`（默认 0.4）

**验证：** 单元测试覆盖三种占比（0、0.2、0.6），确认分类正确；controller 集成测试确认局部分类时选中 LOCAL_EDITING。

**原则：** proxy Policy 与帧占比规则结合——Policy 提供学习信号，启发式提供可解释兜底，贴合你的原始直觉。决策记录同时保存 Policy 原始动作和启发式最终动作，供后续 Trial 审计。

### 任务 4（G4）：videophy2 全链路入口

**文件：** 新增 `agents/wanphysics/run_videophy2_loop.py`

- CLI：`--manifest`（默认 `evaluation/manifests/videophy2_pilot300.json`）或 `--csv`；`--prompt-field {caption|upsampled_caption}`（默认 caption）；`--limit N`；`--task-id-field sample_id`
- 从 manifest/CSV 读取 prompt 列表，逐条：
  - 复用 `run_loop.py` 的 `LoopController + _VllmHandoffCritic + ExecutorRegistry` 组装（抽成共享 `build_loop(...)` helper，避免 DRY 违规）
  - 输出到 `outputs/videophy2_run_时间/{sample_id}/`
  - 每条写 `loop_result.json`（含 stop_reason、best_physics_score、detector_backend、错误范围判定轨迹）和 `trials.jsonl`
- 汇总写 `outputs/videophy2_run_时间/summary.json`：达标率、平均轮数、全局/局部修复各触发次数

**验证：** 用 `--limit 2 --prompt-field caption` 做无 GPU dry-run（mock generator/critic）确认链路装配正确；服务器上用 `--limit 1` 跑真实 GPU smoke。

**原则（DRY）：** run_loop.py 与 batch_run_loop.py 与本入口共享 `build_loop()`，不复制组装逻辑。

## 六、边界与非目标

- **不做真实 Trial 训练/切 value-led**：本方案用现有 proxy Policy + 帧占比启发式即可跑通闭环；真实 Trial 采集（run_trial_campaign.py）和重训（retrain_policy.py）已在第二次对齐中就位，属独立后续。
- **不改 candidate_id 策略、不改绝对路径**：与本次串链路目标无关，维持现状。
- **SAM mask 落盘只覆盖 critical_frames**：不为全部帧生成 mask，控制开销（KISS/YAGNI）。
- **security/资源**：Wan2.2 + vLLM 在 40GB A100 上顺序交接，P0-6 已解决轮间显存释放；videophy2 批量跑需注意逐条串行，避免并发 OOM。
