## 1. 当前一句话结论

服务器端 V2 现在具备完整闭环代码路径：

```text
VideoPhy2 manifest / prompt
  -> Wan2.2 生成候选视频
  -> SAM2 + Qwen3-VL Critic 评估
  -> CriticReport V2 codec 无损解析
  -> mask_manifest.json / full critic_report 落盘
  -> Acceptance Gate + Policy Guard
  -> Repair Policy 决策
  -> 四动作 Executor：Prompt Repair / Global Regeneration / Local Editing / Reject
  -> 执行动作后 Re-Critic / Re-Gate
  -> RepairTrace + WanRepairTrialV2 + run summary
```

本次真实一条样本 smoke 已成功跑通：

```text
Wan initial generation
  -> SAM2/VLM Critic
  -> forced Prompt Repair Executor
  -> Wan repaired generation
  -> Re-Critic/Re-Gate
  -> loop_result / repair_trace / trials / summary
```

Local Editing 分支由于本次真实样本首轮没有局部 violation，因此没有在这条样本中触发 ProPainter；但 V2 strict ProPainter backend 已单独用真实 smoke 验证通过：

```text
mask_manifest.json + per-frame masks
  -> StrictProPainterLocalEditor
  -> ProPainter inference
  -> edited mp4
```

## 3. V2 代码结构总览

### 3.1 入口脚本层

```text
agents/wanphysics/run_videophy2_loop_v2.py
agents/wanphysics/run_actual_trials_v2.py
agents/wanphysics/gen_step.py
agents/wanphysics/eval_step.py
```

| 文件 | 角色 |
|---|---|
| `run_videophy2_loop_v2.py` | V2 主入口，从 VideoPhy2 manifest 取样本，装配真实后端，运行 repair loop |
| `run_actual_trials_v2.py` | forced four-action trial 入口，对同一样本分别强制四动作，收集 `WanRepairTrialV2` |
| `gen_step.py` | Wan 生成子进程入口，被 `WanSubprocessGenerator` 调用 |
| `eval_step.py` | Critic 子进程入口，被 `V2SubprocessCritic` 调用 |

### 3.2 V2 sidecar 核心模块

```text
generators/wanphysics/v2/
  artifacts.py
  build_backends.py
  checkpoint_gate.py
  critic_backend.py
  critic_codec.py
  critic_profiles.py
  executors.py
  guardrails.py
  mask_manifest.py
  memory_adapter.py
  policy_guard.py
  preflight.py
  prompt_renderer.py
  propainter_strict_editor.py
  resource_coordinator.py
  runner.py
  scorers_semantic.py
  trials.py
```

| 文件 | 核心职责 |
|---|---|
| `build_backends.py` | 把 generator / critic / executor / policy / artifacts 装配成 runner |
| `runner.py` | V2 状态机：Generate → Critic → Gate → Decide → Execute → Re-Critic |
| `critic_backend.py` | 调用 `eval_step.py` 子进程并拿回 critic payload |
| `critic_codec.py` | 无损恢复 CriticReport，保留 violations / critical_frames / evidence |
| `mask_manifest.py` | 把 `sam2_masks/*.png` 汇总为可校验、可追溯的 mask manifest |
| `propainter_strict_editor.py` | V2 strict ProPainter editor，逐帧读取 manifest mask，禁止 fallback |
| `executors.py` | 四动作 executor：Prompt Repair / Global Regeneration / Local Editing / Reject |
| `policy_guard.py` | scope + capability 裁决，决定最终动作 |
| `guardrails.py` | 接受门，记录 physics / confidence / coverage / semantic / quality |
| `preflight.py` | 运行前就绪检查，决定 capability mask |
| `trials.py` | `WanRepairTrialV2` schema，保存真实 before/action/after trial |
| `artifacts.py` | 统一落盘 run manifest、critic report、trace、trial、summary、status |
| `checkpoint_gate.py` | proxy checkpoint 加载门禁 |
| `memory_adapter.py` | proxy memory 状态识别，默认 disabled |
| `resource_coordinator.py` | vLLM owner/PID 逻辑，避免宽泛 kill |
| `prompt_renderer.py` | 把 critic evidence 渲染成确定性修复 prompt |
| `scorers_semantic.py` | semantic scorer，默认关闭，按配置 shadow 记录 |

### 3.3 共享契约层

```text

src/physgenloop/learning_repair/base_contracts.py
```

## 4. V2 运行流程详解

### 4.1 总流程图

```text

[run_videophy2_loop_v2.py]

        +--> preflight: ProPainter / ffmpeg / cv2 / GPU / vLLM port
        +--> checkpoint_gate: proxy checkpoint 授权
        +--> memory_adapter: proxy memory status
        +--> generator: WanSubprocessGenerator
        +--> critic: _V2Critic(V2SubprocessCritic + artifacts)
        +--> executors: four-action ExecutorRegistry
        |
        v
[ActionAwareRunnerV2]
        |
        v
Generate current candidate by Wan2.2
        |
        v
Critic evaluate by SAM2 + Qwen3-VL
        |
        +--> critic_report.json
        +--> critic.json
        +--> mask_manifest.json
        +--> resource_metrics.jsonl
        |
        v

        |
        v

        |
        +--> prompt_repair
        +--> global_regeneration
        +--> local_editing
        +--> reject
        |

        |
        v

        |
        v

        |
```

入口：`agents/wanphysics/run_videophy2_loop_v2.py`


脚本读取 `sample_id`、`prompt` 或 `caption`，并由 `--limit` 控制样本数。

检查项：

### 4.4 checkpoint gate

当前 policy 是 proxy research checkpoint，因此真实 GPU 运行必须显式传：

```bash
--allow-proxy-policy
```

否则入口会退出，不加载 proxy checkpoint。

### 4.5 Wan2.2 生成

调用链：

```text
ActionAwareRunnerV2
  -> WanSubprocessGenerator.generate()
  -> agents/wanphysics/gen_step.py
  -> generators/wanphysics/wan_generator.py
  -> Wan2.2 model
```

当前 V2 默认：

```yaml
generator:
  num_frames: 81
  height: 480
  width: 832
  fps: 24
runtime:
  gpu_mode: dual_gpu
  generator_gpu: 0
  critic_gpu: 1
```

即 Wan2.2 走 GPU0，vLLM/Qwen3-VL 走 GPU1。

### 4.6 SAM2 + Qwen3-VL Critic

调用链：

```text
ActionAwareRunnerV2
  -> _V2Critic.evaluate()
  -> V2SubprocessCritic.evaluate()
  -> agents/wanphysics/eval_step.py
  -> SAM2 video predictor + vLLM/Qwen3-VL
  -> raw critic payload
  -> critic_codec.py
  -> CriticReport
```

本次 smoke 日志中出现：

```text
cannot import name '_C' from 'sam2'
Skipping the post-processing step
```

这是 SAM2 可选 C++/CUDA post-processing 扩展缺失 warning。本次 run 未失败，且 `critic_degraded=false`。

### 4.7 CriticReport codec

代码：`generators/wanphysics/v2/critic_codec.py`

V2 需要保留：

- `decision`；
- `is_physical`；
- `physics_score`；
- `confidence`；
- `coverage`；
- `violations`；
- `critical_frames`；
- `evidence`；
- `mask_uri` / `mask_uris`。

这是为了避免旧链路中“子进程 critic 输出有 violations，但主进程反序列化后丢失”的问题。

### 4.8 mask manifest

代码：`generators/wanphysics/v2/mask_manifest.py`

来源：

```text
candidate_dir/sam2_masks/{object}_{frame:05d}.png
```

输出：

```text
candidate_dir/mask_manifest.json
```

严格规则：

- 缺 mask：invalid；
- 空 mask：invalid；
- 几乎全帧 mask：invalid；
- 尺寸不一致：invalid；
- frame 越界：invalid；
- SHA 不一致：invalid。

本次 smoke 后已修正一个细节：只要存在 requested mask 缺失，manifest status 会标为 `invalid`，不再把 partial valid mask 误标成完全 `ok`。

### 4.9 Acceptance Gate

代码：`generators/wanphysics/v2/guardrails.py`

当前配置：

```yaml
acceptance:
  mode: shadow
  physics_score: 0.80
  confidence: 0.60
  coverage: 0.50
  semantic_score: 0.85
  quality_score: 0.75
```

解释：

- `shadow`：记录阈值和 side score，但主要用于审计；
- `enforce`：才把这些阈值变成硬门禁；
- 当前 semantic scorer 默认关闭，因此 `semantic_score=null`。

### 4.10 Policy + PolicyGuard

代码：

```text
generators/wanphysics/v2/policy_guard.py
generators/wanphysics/repairer.py
```

流程：

1. policy 根据 CriticReport 输出 proposed action；
2. `policy_guard.resolve_action()` 根据 scope、mask、capability 做最终动作裁决；
3. 写入 `repair_decision.json` 和 `repair_trace.jsonl`。

local action 只有在以下条件全部满足时才允许：

```text
local_editing capability == true
mask_manifest strict verification passed
scope == local
```

### 4.11 四动作 Executor

代码：`generators/wanphysics/v2/executors.py`

| 动作 | Executor | 行为 |
|---|---|---|
| `prompt_repair` | `DecisionPromptRepairExecutor` | 使用 critic evidence 渲染修复 prompt，再调用 Wan 重新生成 |
| `global_regeneration` | `OriginalPromptGlobalRegenerationExecutor` | 用原始 prompt 全局重生成 |
| `local_editing` | `MaskSequenceLocalEditingExecutor` | 使用 StrictProPainterLocalEditor 做局部 ProPainter 修复 |
| `reject` | `AuditedRejectExecutor` | 从历史候选中选择最终候选并终止 |

重要点：

- Prompt executor 不会二次调用 policy；
- Local executor 必须有 `LocalEditTarget`；
- Local target 的 `mask_uri` 必须指向 `mask_manifest.json`；
- Executor 执行后必须返回 `ExecutionResult`。

### 4.12 StrictProPainterLocalEditor

代码：`generators/wanphysics/v2/propainter_strict_editor.py`

只用于 V2；旧 `generators/wanphysics/local_editor.py` 保留不删。

Strict editor 规则：

```text
LocalEditTarget.mask_uri == mask_manifest.json
critical_frames 非空
object 能在 manifest 中找到
每个 critical frame 必须有有效 mask
mask SHA 必须匹配
mask 尺寸必须匹配视频尺寸
mask 不能空
mask 不能几乎全帧
不允许全白 fallback
不允许复制第一张 mask 到所有帧
```

ProPainter 输出兼容：

- 优先接受官方输出 `inpaint_out.mp4`；
- 如果没有 mp4，则递归读取 png/jpg 帧重新编码。

### 4.13 Re-Critic / Re-Gate

代码：`generators/wanphysics/v2/runner.py`

动作执行后：

1. executor 产出的 after candidate 成为 current；
2. 立即再次调用同一个 Critic；
3. 计算 after physics / after gate；
4. 写入 loop result；
5. 若非 force 模式且 after accepted，则终止；否则进入下一轮。

### 4.14 WanRepairTrialV2

代码：`generators/wanphysics/v2/trials.py`

输出：

```text
sample_dir/trials.jsonl
schema_version = wan-repair-trial/2.0
```

重要字段：

- source candidate；
- prompt；
- critic_before；
- decision；
- execution；
- before_scores；
- critic_after；
- after_scores；
- successful；
- failure_reason；
- generator provenance；
- critic profile；
- policy mode；
- research_only。

V2 不把 Wan trial 伪装成 Hunyuan/Blender canonical trial。只有团队批准映射后，才允许调用 canonical adapter。

### 5.1 smoke 命令

最终复核使用的是“当前代码版”一条样本 smoke。所有输出均放在 `worklog/2026_07_21/` 下：

```bash

PIDFILE=$WORKLOG/physgenloop_v2_full_chain_smoke_one_sample_current_$TS.pid

mkdir -p "$RUN_OUT"

nohup bash -c '
  echo "[SMOKE] start $(date -Iseconds)"
  cd /root/PhysGenLoop- || exit 2
  PYTHONPATH=/root/PhysGenLoop-:/root/PhysGenLoop-/src \
    envs/main/bin/python agents/wanphysics/run_videophy2_loop_v2.py \
      --enable \
      --allow-proxy-policy \

      --force-action prompt_repair \
      --output-root "$1"
  status=$?
  echo "[SMOKE] end $(date -Iseconds) exit_status=$status"
  echo "$status" > "$2"
  exit $status
' smoke_job "$RUN_OUT" "$STATUS" > "$LOG" 2>&1 &

echo $! > "$PIDFILE"
```

### 5.2 为什么使用 `--force-action prompt_repair`

本次 smoke 目的是验证真实执行闭环。若不 force，首轮 candidate 被判 accepted 时 runner 可能直接结束，无法覆盖 executor 和 re-critic。

使用 `--force-action prompt_repair` 后：

- 仍然跑真实 Wan 初始生成；
- 仍然跑真实 SAM2/VLM Critic；
- policy 仍被加载并记录 proposed action；
- 最终动作强制为 Prompt Repair；
- executor 必须生成 after candidate；
- after candidate 必须再次经过 Re-Critic/Re-Gate；
- 必须产出 `WanRepairTrialV2`。

```bash
import cv2
import numpy as np
from physgenloop.contracts import GeneratedCandidate
from physgenloop.learning_repair.contracts import LocalEditTarget
from generators.wanphysics.v2.mask_manifest import MaskFrame, MaskManifest, MaskObject

out_dir = root / 'output'
```

```text
v2_run_YYYYMMDD_HHMMSS/
  run_manifest.json

    {candidate_id}/

      raw_payloads/
      sam2_masks/
        {object}_{frame:05d}.png
```

关键产物含义：

| 文件 | 含义 |
|---|---|
| `run_manifest.json` | run id、sample id、分辨率、GPU 模式、preflight、force action |
| `checkpoint_gate.json` | checkpoint 是否允许加载、proxy/deployment 状态 |
| `memory_status.json` | proxy memory 是否启用、是否可解析 |
| `critic_report.json` | 完整 CriticReport，包含 violations / critical_frames / evidence |
| `mask_manifest.json` | 严格 mask 证据表，含 sha / size / ratio / valid / reason |
| `repair_trace.jsonl` | 动作级审计，记录 policy action、final action、executor 状态 |
| `loop_result.json` | 状态机级结果，含 before/after candidate 和 re-gate |
| `trials.jsonl` | `WanRepairTrialV2` 训练样本 |

## 10. 当前注意事项

### 10.1 forced prompt repair 可能产生负样本

本次初始 candidate 物理分数为 1.0，forced prompt repair 后降到 0.177。因为这次是强制动作验证，不代表 policy 自动选择一定会这样。

它说明：

- executor 成功；
- re-critic 成功；
- after 不如 before 时，trial 会记录 `successful=false`；
- 这类负样本可用于后续 action-value 学习。

### 10.2 SAM2 `_C` warning

当前 SAM2 可选 C++/CUDA post-processing 扩展不可用，但 smoke 没失败。若后续要更严格，建议补 SAM2 扩展编译，或把该 warning 显式纳入 degraded critic 判定。

### 10.3 Local Editing 是否触发取决于严格 mask

即使 `local_editing.enabled=true`，最终是否允许 local action 仍取决于：

```text
preflight.local_editing == true
and mask_manifest strict verification passed
and scope == local
```

这避免把不完整 mask 输入 ProPainter。

### 10.4 当前 policy 是 proxy research

真实 GPU V2 运行必须显式：

```bash
--allow-proxy-policy
```

不要把当前 proxy checkpoint 结果描述为 deployment-ready。
